# Documentation archive

This directory stores historical handoffs, investigation notes, proposals, and migration plans that are useful for provenance but are not current user-facing guidance.

Rules:

- Do not link to archive files as the primary source of truth for current behavior.
- New task-specific agent reports, generated audits, dependency trees, or release checklists should not be added here by default; keep them in `.omx/` or regenerate them locally.
- If historical content becomes current again, extract the durable guidance into a current doc and leave the original artifact archived.
