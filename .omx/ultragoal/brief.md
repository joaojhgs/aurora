## Requirements
- Keep WebRTC signaling/session IDs separate from stable mesh peer IDs.
- Use stable mesh IDs for manifests, peer registry records, saved inbound credential lookup/persistence, ACL/policy-facing diagnostics, and reconnect flows.
- Keep session IDs for MQTT signaling and DataChannel transport addressing.
- Preserve backward compatibility for existing single-token or session-keyed saved credentials.

## Implementation Steps
1. Add explicit local stable identity support to `RTCClient`: retain `_peer_id` as signaling ID, add stable mesh peer ID/node name setters/getters, and helper mapping from session ID to stable ID.
2. Update auth, pairing, manifest, registry, ping/pong, capacity, disconnect, and diagnostic paths to use stable peer IDs for mesh state while using session IDs for DataChannel sends.
3. Make saved-token lookup prefer known remote stable ID, fall back to `_default`/single legacy token/session-keyed values, and migrate learned stable IDs into `_saved_auth_tokens`.
4. Ensure incoming manifest `peer_id` drives the stable ID mapping and registry update, including distinct session-vs-stable reconnect behavior.
5. Add focused unit tests covering stable local manifest ID, distinct session/stable remote registration, and saved-token fallback/rekey on reconnect.
6. Run targeted gateway tests and the mesh baseline subset that fits the changed surface.

## Acceptance Criteria
- Manifest generated by RTCClient advertises stable local peer ID, not the WebRTC signaling UUID.
- Remote peer registry and Auth persistence callbacks receive manifest/auth stable IDs, not session IDs.
- Reconnect with a different signaling ID can reuse credentials keyed by stable remote peer ID.
- Existing legacy/default credential behavior still works for migration.
- Targeted tests pass.

## Risks
- Transport sends still require session IDs; changing all peer keys to stable IDs would break DataChannel routing. Mitigation: maintain mapping helpers and only translate at mesh/policy boundaries.
- Remote stable ID is unknown before auth/manifest. Mitigation: use session ID until learned, then re-register/update stable mesh state and preserve legacy fallback token lookup.
